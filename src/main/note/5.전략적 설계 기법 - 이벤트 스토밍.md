5.전략적 설계 기법 - 이벤트 스토밍
=========================
## ⭕ 이벤트 스토밍
이벤트 스토밍이란 비즈니스 프로세스를 이해하고 시각화하기 위해 도메인 전문가와 개발팀이 함께 협업하는 워크숍 기법입니다.

### ◉ 이벤트 스토밍 준비
이해 관리자가 모두 한자리에 모여서 포스트잇에 비즈니스에서 발생하는 중요한 사건들을 작성하여 비지니스 흐름대로 대시보드에 붙인다.
특정 색깔 포스트잇에는 도메인 이벤트를 또다른 색깔에는 명령을 다른 색깔에는 행위자를 적어서 시간 순서대로 정렬하며 전체 프로세스를 시각화하고 문제점은 별도 색깔로 표시하여 모든 참여자가 비즈니스를 동일하게 이해할 수 있도록 한다.

### ◉ 이벤트 스토밍 구성요소
1. 도메인 이벤트
  - 시스템에서 발생하는 중요한 이벤트 (e.g. 주문생성됨, 결제가 요청됨)
  - 데이터가 아닌 비지니스 프로세스에 집중
  - 과거형 동사로 표현
2. 핫스팟(Hotspot)
  - 질문, 가정, 경고, 의견 수렴이 필요한 내용
  - 병목 구간, 자동화 필요한 수작업, 도메인 지식이 없는 경우
  - 완전히 정의되지 않은 영역, 해결해야하는 문제
3. 커맨드(Command)
  - 이벤트를 트리거하는 명령 (e.g. 주문하기, 결제하기)
  - 현재 동사로 표현
4. 액터(Actor)
  - 커맨드를 동작하게 하는 사용자/역할 (e.g. 고객, 관리자)
  - 사람이나 역할의 이름을 명사로 표현
5. 폴리시(Policy)
  - 자동화 정책: 이벤트나 커맨드를 시작하는 시나리오 (e.g. 주문이 완료되면 배송 요청)
  - 커맨드는 특정 도메인 이벤트가 바랭할 때 자동 실행된다.
6. 외부 시스템(External System)
  - 연계가 필요한 외부 시스템 또는 프로세스 (e.g. 결제 게이트웨이, 물류 시스템)
  - 명사로 표현
7. 애그리거트(Aggregate)
  - 도메인 이벤트와 커맨드를 그룹화하여 일관된 상태를 유지하는 단위 (e.g 주문, 결제, 배송)
  - 비즈니스 규칙을 적용하고 상태를 관리하는 객체
  - 명사로 표현
